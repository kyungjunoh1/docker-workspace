### 명령어 정리

- docker search mysql : mysql 이미지 확인
- docker pull mysql : mysql 이미지 최선버전 다운로드
- docker images : 내 docker-engine에 이미지 확인
- docker pull mysql:8.4
- docker images
- docker rmi mysql:8.4 : 이미지 삭제( 이름 )
- docker images
- docker rmi e5dc14f6e01c : id로 이미지 삭제
- docker images : 삭제 확인
- docker pull mysql:8.4 : 다시 이미지 다운로드
- docker images : 다운로드 확인
- docker ps : 실행중인 컨테이너 확인( 이미지 실행 = 컨테이너 )
- docker ps -a : -a ( 실행중이거나, 멈춰있거나 등등의 모든 컨테이너 확인 )
- docker images : 실행할 이미지 확인
- docker run a2d126916bc2 : 이미지 id로 이미지 실행( 컨테이너 화 - 에러 발생 )
- docker ps : 실행 중인 컨테이너 확인( 없음 )
- docker ps -a : 위 이미지 실행시 에러 발생으로 생성된 컨테이너 존재( status : exited)
- docker images : 실행할 이미지 이름 확인
- docker run --name test-mysql -e MYSQL_ROOT_PASSWORD=1234 mysql:8.4 : name-컨테이너 이름, -e - 환경변수 설정. 기본 mysql은 구동시 비밀번호 설정이 필요하다.
- docker run -d --name test-mysql2 -e MYSQL_ROOT_PASSWORD=1234 mysql:8.4 : -d = 백그라운드 실행
- docker ps : 구동 중인 컨테이너 확인
- docker ps -a : 모든 컨테이너 확인
- docker images : 삭제 이미지 확인
- docker rmi a2d126916bc2 : 삭제 실패( 컨테이너가 존재하는 경우 삭제 안됨 )
- docker ps -a : 모든 컨테이너 확인
- docker rm happy_spence : 멈춰있는 컨테이너 삭제( 성공 )
- docker ps -a : 삭제 성공 확인
- docker rm 45259c334395 : 실행중인 컨테이너(id적용) 삭제 ( 실패 )
- docker stop test-mysql2 : 컨테이너(names활용) 중지
- docker ps -a : 중지 확인( status : exited )
- docker rm -f test-mysql2 test-mysql : -f 강제 삭제( 실행중인 컨테이너 )
- docker images : 삭제할 이미지 확인
- docker rmi a2d126916bc2 : 이미지 삭제
